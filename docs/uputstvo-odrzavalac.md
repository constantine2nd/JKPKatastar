## 1. Увод

Ово упутство намењено је одржаваоцима (MAINTAINER) JKP Катастар система. Одржавалац је техничка улога намењена за масовни увоз, извоз и управљање подацима у систему.

**Одржавалац може:**
- Увозити податке у базу путем JSON фајлова (увоз у великом обиму)
- Извозити целе колекције из базе у JSON фајл (спреман за поновни увоз)
- Све функције службеника (управљање гробљима, гробним местима, покојницима, платиоцима, захтевима)
- Управљати корисничким налозима (прегледати, креирати, уређивати, брисати)

**Важна напомена о улози ОДРЖАВАОЦА:**
> Улога MAINTAINER **не може** бити додељена од стране администратора — то је намерна безбедносна мера. Одржавалац може доделити ову улогу другом кориснику путем табеле управљања корисницима. Прва додела иде преко администратора базе података.

## 2. Пријава у систем

Кликните **„Пријава"** у горњем десном углу. Унесите своју e-mail адресу и лозинку.

> **Напомена о безбедности:** Налог одржаваоца омогућава потпун приступ свим подацима и функцијама за увоз. Чувајте своје приступне податке на сигурном месту.

### Заборављена лозинка

Кликните **„Заборавили сте лозинку?"** на страници за пријаву. Унесите своју e-mail адресу — систем ће послати линк за ресетовање.

## 3. Увоз података

Из главног менија изаберите **„Увоз података"** (доступно само одржаваоцима).

### Покретање чаробњака за увоз

Увоз се одвија у три корака:

#### Корак 1 — Избор колекције

Из падајућег менија изаберите у коју колекцију желите да увезете податке:

| Колекција | Опис |
|---|---|
| **Гробља** | Списак гробаља |
| **Врсте гробних места** | Типови гробних места са капацитетима |
| **Гробна места** | Сва гробна места са локацијама |
| **Покојници** | Евиденција покојника |
| **Платиоци** | Евиденција платилаца |
| **Корисници** | Кориснички налози |

Кликните **„Следећи"** за наставак.

#### Корак 2 — Учитавање фајла и преглед

Кликните на зону за учитавање и изаберите JSON фајл са подацима. Систем ће аутоматски учитати и приказати преглед првих 10 записа у табели.

**Формат фајла:**

Фајл мора бити у MongoDB Extended JSON формату — низ објеката:

```json
[
  {
    "_id": { "$oid": "60c72b2f9b1d8a3b4c8e4f2a" },
    "name": "Назив",
    "createdAt": { "$date": "2024-01-15T00:00:00.000Z" }
  }
]
```

Поља са `$oid` биће аутоматски конвертована у MongoDB ObjectId, а поља са `$date` у Date вредности.

> **Важно:** Записи са ID-јем који већ постоји у бази биће **прескочени** (без грешке). Нови записи биће додати. Постојећи записи неће бити измењени.

Кликните **„Следећи"** за покретање увоза.

#### Корак 3 — Резултат увоза

По завршетку система приказаће резултат:

| Стање | Значење |
|---|---|
| **Увезено X од Y** | X записа је успешно увезено, Y је укупан број у фајлу |
| Делимичан успех | Неки записи прескочени (дупликати или грешке у формату) |

Кликните **„Ресетуј"** да покренете нови увоз.

### Препоруке за увоз

- Увек прво тестирајте увоз са малим бројем записа
- Пре увоза гробних места, увезите гробља и врсте гробних места на које се позивају
- Пре увоза покојника и платилаца, увезите гробна места на која се позивају
- Чувајте архиву оригиналних фајлова за увоз

## 4. Извоз података

Из главног менија (група **Одржавалац**) изаберите **„Извоз података"**.

### Покретање извоза

1. Из падајућег менија изаберите колекцију коју желите да извезете
2. Кликните **„Извези"**
3. Фајл се аутоматски преузима под именом `<колекција>-export.json`

### Формат извезеног фајла

Фајл је у MongoDB Extended JSON формату — исти формат који чаробњак за увоз прихвата:

```json
[
  {
    "_id": { "$oid": "60c72b2f9b1d8a3b4c8e4f2a" },
    "name": "Назив",
    "createdAt": { "$date": "2024-01-15T00:00:00.000Z" }
  }
]
```

Извезени фајл може се директно употребити за:
- Прављење резервне копије одређене колекције
- Пренос података у другу инсталацију система
- Увоз назад у исту или другу базу путем чаробњака за увоз

## 5. Управљање корисницима

Из главног менија изаберите **„Управљање корисницима"**.

### Табела корисника

Приказује све кориснике у систему са следећим колонама:

| Колона | Опис |
|---|---|
| Имe | Пуно имe корисника |
| E-mail | E-mail адреса (корисничко имe) |
| Улога | ПОСЕТИЛАЦ / СЛУЖБЕНИК / АДМИНИСТРАТОР |
| Активан | Да / Не — да ли налог може да приступа систему |
| Верификован | Да / Не — да ли је e-mail потврђен |

> **Напомена:** Само одржавалац може додати улогу **ОДРЖАВАЛАЦ** другом кориснику путем ове табеле. Администратор нема ту могућност.

### Уређивање налога

Кликните иконицу **оловке** поред жељеног корисника да промените улогу, статус активности или верификацију.

### Деактивација и брисање

- **Деактивација** — Поставите **„Активан"** на „Не". Корисник не може да се пријави, али подаци остају.
- **Брисање** — Кликните иконицу корпе. Трајно и не може се поништити; препоручљиво је деактивирати уместо брисати.

## 6. Функције службеника

Одржавалац има приступ свим функцијама описаним у **Упутству за службенике**, укључујући:

- Управљање гробним местима (додавање, уређивање, брисање)
- Управљање покојницима по гробном месту
- Управљање платиоцима по гробном месту
- Управљање гробљима (додавање, уређивање, брисање)
- Управљање врстама гробних места
- Обрада захтева грађана (прихватање / одбијање)
- Претрага покојника
- Генерисање PDF извештаја

За детаљна упутства о овим функцијама погледајте **Упутство за службенике**.

## 8. Препоруке за одржаваоца

### Безбедност

- Налог одржаваоца омогућава масовни упис у базу — чувајте га посебно пажљиво
- Не делите налог са другима — свака особа треба свој налог
- После завршетка миграције података, размотрите привремену деактивацију налога

### Редослед увоза

При иницијалном пуњењу базе поштујте следећи редослед:

1. Гробља
2. Врсте гробних места
3. Гробна места
4. Покојници
5. Платиоци

### Верификација после увоза

После сваког увоза проверите резултат у одговарајућој табели у систему (нпр. отворите „Преглед гробних места" и проверите да ли су подаци исправни).
